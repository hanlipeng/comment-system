# 功能需求: 抽奖留言板系统

## 元数据
- **ID**: REQ-MACRO-001
- **类型**: Macro (宏观)
- **负责人**: User

## 用户故事 (User Story)
1. **普通用户 (Web端)**:
    - 我希望访问一个网页，能够针对特定的活动提交留言（需填写手机号以便领奖）。
    - 我希望在活动结束后，能在网页上看到开奖结果。
2. **管理员 (Rust GUI端)**:
    - 我希望有一个桌面应用来管理多个独立的活动。
    - 我希望能查看某个活动下的所有留言列表。
    - 我希望有一个按钮能从留言池中随机抽取一条作为幸运留言。
    - 我希望通过这个应用作为主机服务器（控制 Web 服务器的启动/停止）。

## 功能需求 (Functional Requirements)
1. **服务器模块 (Server Module)**:
    - 处理来自 Web 端的 HTTP 请求 (POST 提交留言, GET 获取状态/结果)。
    - 存储数据 (活动信息, 留言数据)。
    - 为管理端提供 API (或如果是嵌入式架构则直接访问)。
2. **管理端 (Admin Client - Tauri)**:
    - 创建/删除活动的界面。
    - 查看留言列表的界面。
    - 触发随机抽奖逻辑的界面。
    - 控制 HTTP 服务器生命周期 (启动/停止)。
3. **Web 客户端 (Web Client)**:
    - 适配移动端的用户界面。
    - 提交表单：包含昵称、留言内容、**手机号**。
    - 轮询或 WebSocket 机制以检查活动状态/开奖结果。

## 验收标准 (Acceptance Criteria)
- [ ] 管理员可以通过 GUI 启动服务器。
- [ ] 用户可以通过 Web 端提交留言（含手机号）。
- [ ] 管理员可以在 GUI 中看到提交的留言。
- [ ] 管理员可以执行抽奖，系统选中一名获奖者。
- [ ] 用户可以在 Web 端看到中奖留言。
